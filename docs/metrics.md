# Metrics & Triggers

## Web Login Scenario

The web scenario runs every **5 minutes** from the Zabbix Server. It performs two steps using a shared cookie jar — no scripts required.

| Step | Method | URL | Check |
|------|--------|-----|-------|
| Load portal | GET | `https://{HOST.CONN}:{$OVPN_WEB_PORT}/` | HTTP 200 + body contains `OpenVPN CWS` |
| Authenticate LDAP user | POST | `https://{HOST.CONN}:{$OVPN_WEB_PORT}/__auth__` | HTTP 200 (failure returns 403) |

Step 2 sends `username` and `password` as form data with headers `X-OpenVPN: 1`, `X-CWS-Proto-Ver: 2`, and `Referer`. Zabbix handles the session cookie (`openvpn_sess_*`) automatically between steps.

**How failures surface:** a failed login returns HTTP 403, which does not match `status_codes: 200` → step fails → `web.test.fail` becomes non-zero → trigger fires.

### Web Scenario Items (auto-generated by Zabbix)

| Item | Key | Description |
|------|-----|-------------|
| Failed step | `web.test.fail[OpenVPN AS: LDAP login test]` | Step number that failed; `0` = all OK |
| Last error message | `web.test.error[OpenVPN AS: LDAP login test]` | Error text from last failure |
| Response code — Load portal | `web.test.rspcode[…,Load portal]` | HTTP status code |
| Response code — Authenticate | `web.test.rspcode[…,Authenticate LDAP user]` | HTTP status code |
| Response time — Load portal | `web.test.time[…,Load portal,resp]` | Seconds |
| Response time — Authenticate | `web.test.time[…,Authenticate LDAP user,resp]` | Seconds |
| Download speed — Load portal | `web.test.in[…,Load portal,bps]` | Bytes per second |
| Download speed — Authenticate | `web.test.in[…,Authenticate LDAP user,bps]` | Bytes per second |

> `web.test.error` retains the last error message even after recovery. The error trigger uses `AND web.test.fail > 0` to prevent false positives from stale error text.

### Web Scenario Triggers

| Trigger | Severity | Condition | Depends on |
|---------|----------|-----------|------------|
| Web login test not running | Average | `nodata(15m) = 1` | — |
| Web login test failed | High | `fail > 0` for 2 consecutive checks | not running |
| Web login test error | High | `error` has content AND `fail > 0` | failed |
| Web login slow response (>5s) | Warning | Auth step > 5s for 5 min | not running |

**Dependency chain** — suppresses noise when the root cause is connectivity:
```
not running (AVG)
    ├── failed (HIGH)     ← suppressed while test isn't running
    │     └── error (HIGH)  ← suppressed while failed already fires (dedup)
    └── slow (WARN)       ← suppressed while test isn't running
```

**Detection time:** ≤ 5 minutes (one scenario interval).

## External Check Template Metrics

> Requires `openvpn_as_check.py` on the Zabbix Server/Proxy and port 943 reachable. If XMLRPC is not available, disable the master item `OpenVPN AS: Collect all metrics` on the host — the web scenario runs independently.
>
> **Note:** The XMLRPC metrics path has not been tested against a live OpenVPN AS instance. The code is based on the public XMLRPC API documentation. Please report any issues.

| Metric | Item key | Type | Description |
|--------|----------|------|-------------|
| Active sessions | `ovpn.active_sessions` | Integer | Connected VPN clients |
| Bytes in | `ovpn.bytes_in` | Integer (B/s) | Inbound traffic rate |
| Bytes out | `ovpn.bytes_out` | Integer (B/s) | Outbound traffic rate |
| Server version | `ovpn.server_version` | String | OpenVPN AS version string |
| XMLRPC ping | `ovpn.xmlrpc_ping` | Integer (0/1) | XMLRPC API reachability |
| Web portal status | `ovpn.web_portal_status` | Integer (0/1) | Web portal reachability |
| Web portal response | `ovpn.web_portal_response_ms` | Float (ms) | Web portal response time |
| LDAP auth test | `ovpn.ldap_auth_test` | Integer (0/1) | Auth pipeline health via XMLRPC |

All dependent items extract their value from a single master item JSON via JSONPath:

```json
{
  "xmlrpc_ping": 1,
  "active_sessions": 42,
  "bytes_in": 1234567890,
  "bytes_out": 9876543210,
  "server_version": "2.13.1",
  "web_portal_status": 1,
  "web_portal_response_ms": 142.5,
  "ldap_auth_test": 1
}
```

When a metric fails to collect, its value is `null` — the dependent item discards the value and retains the previous reading.

### External Check Triggers

| Trigger | Severity | Condition |
|---------|----------|-----------|
| XMLRPC API is unreachable | High | `xmlrpc_ping = 0` for last 3 checks |
| Web portal is unreachable | High | `web_portal_status = 0` for last 3 checks |
| Web portal slow response | Warning | Response time > `{$OVPN_WEB_RESPONSE_WARN}` ms for 5 min |
| LDAP authentication is failing | High | `ldap_auth_test = 0` for last 3 checks |
| High session count | Warning | `active_sessions` > `{$OVPN_SESSION_WARN}` |
| Critical session count | High | `active_sessions` > `{$OVPN_SESSION_HIGH}` |
| No data collection | Average | No data received for 5 minutes |

Thresholds for session count and web response time are controlled by macros — see [Configuration](configuration.md).

## Agent Template Metrics

| Metric | Item key | Type | Description |
|--------|----------|------|-------------|
| Process count | `proc.num[openvpnas]` | Integer | Number of running openvpnas processes |
| Service state | `systemd.unit.info[{$OVPN_SERVICE_NAME},ActiveState]` | String | systemd ActiveState value |
| Error log | `log[{$OVPN_LOG_PATH},{$OVPN_LOG_REGEXP},,100,skip]` | Log | Matching lines from error log |

### Agent Template Triggers

| Trigger | Severity | Condition |
|---------|----------|-----------|
| Process is not running | Disaster | `proc.num = 0` for last 3 checks |
| Service is not active | High | `ActiveState ≠ "active"` |
| Error found in log | Warning | Matching line found in error log (manual close) |
